# 内容制作平台工作流程

## 平台定位

### 第一梯队（主战场）
- **小红书**：种草 + 高客单，图文形式
- **抖音**：放大 + 转化，短视频脚本

### 第二梯队（长期价值）
- **微信**：私域 + 品牌，长图文
- **百家号**：SEO引流，搜索优化

## 工作流程

### 1. 创建新文章

```bash
# 复制模板
cp drafts/template.md drafts/your-post.md
```

### 2. 编写内容

按照模板结构填写：
- **核心问题**：痛点描述 + 背景
- **核心模型**：用模型思维拆解
- **方法迁移**：从其他领域借鉴
- **判断**：给出观点

### 3. 生成平台版本

```bash
cd cygnus.github.io
python scripts/generate.py drafts/your-post.md
```

生成文件：
- `platforms/xiaohongshu/your-post.md`
- `platforms/douyin/your-post.md`
- `platforms/wechat/your-post.md`
- `platforms/baijiahao/your-post.md`
- `platforms/summary.md`

### 4. 人工审核修改

检查各平台版本：
- 小红书：调整标题、标签、图片说明
- 抖音：检查脚本时长、节奏
- 微信：调整排版、推荐链接
- 百家号：优化关键词
- 摘要：控制在300字内

### 5. 发布流程

```bash
# 1. 移动到发布目录
mv drafts/your-post.md _posts/YYYY-MM-DD-your-post.md

# 2. 推送到 GitHub
git add .
git commit -m "post: 发布文章"
git push
```

然后手动复制各平台版本到对应平台发布。

## 内容策略

### 判断标准

写之前问自己：**能否沉淀为长期认知资产？**
- YES → 写博客（主内容）
- NO → 做流量切片 or 不做

### 精力分配

- 技术博客（80%精力）：模型、框架、深度分析
- 流量内容（20%精力）：切片、问题导向

## 平台特征

| 平台 | 流量来源 | 爆发能力 | 转化能力 | 适合做什么 |
|------|---------|---------|---------|-----------|
| 小红书 | 搜索+推荐 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 种草/高客单 |
